Privacy Policy

隐私政策

最后更新：2026 年 5 月 10 日

一句话总结

OneStyle 在你的本地浏览器中运行，不会收集你的个人信息，不会上传任何数据到我们的服务器，也不会与任何第三方共享数据。所有标注、评论、设计修改都只保存在你自己的设备上。

1. 我们不收集哪些数据

OneStyle 是一款本地运行的网页可视化样式标注与编辑工具，遵循"最小数据原则"。具体而言：

  • 不要求账号注册或登录
  • 不内置任何分析、统计或广告 SDK
  • 不向任何服务器发送数据（包括我们自己的服务器）
  • 不与任何第三方共享数据
  • 不收集你的个人身份信息（PII）
  • 不访问与扩展功能无关的网页内容

2. 扩展在本地处理哪些数据

为了实现可视化标注与样式编辑功能，扩展会在你主动点击工具栏图标后，在当前标签页中处理以下数据。这些数据全部保留在你本地浏览器中，不会离开你的设备：

页面 DOM 结构与 CSS 样式

用于支持 hover 高亮、元素选中、CSS selector 生成、computed style 读取等功能。读取范围限定于你当前激活了 OneStyle 的标签页，扩展不会在后台或你未交互时访问任何页面。

用户生成的标注数据

包括你修改的元素样式 diff、保存的修改快照、评论文本与图钉位置、自定义颜色库与图标库。这些数据通过 Chrome 提供的 chrome.storage.local API 保存在你本地浏览器中。

3. 数据存储位置

所有数据均存储在你本地浏览器的 chrome.storage.local 与 localStorage 中，不上传至任何远程服务器。

当你卸载扩展、清除浏览器扩展数据，或在 Chrome 设置中清除该扩展的存储时，相关数据将被一并删除。我们无法、也不会以任何方式恢复这些数据。

4. 权限说明

扩展在 manifest 中声明的全部权限及其用途：

activeTab
仅在你主动点击工具栏图标后获得对当前标签页的临时访问权限。扩展不声明任何 host_permissions，不在页面加载时自动注入脚本。
scripting
在你激活扩展时，由 background service worker 通过 chrome.scripting.executeScript 向当前标签页临时注入工具栏 UI 脚本。
storage
在本地保存你的样式修改、评论图钉、自定义颜色与图标库等数据。仅本地存储，不上传。
clipboardWrite
在你点击"复制 Prompt / 复制 JSON"时，将导出的文本写入系统剪贴板。不会读取剪贴板内容。

5. 第三方服务

本扩展不集成任何第三方分析、统计、广告或追踪 SDK，也不与任何第三方服务共享数据。

6. 儿童隐私

本扩展不针对 13 岁以下儿童设计，也不会有意收集任何用户的个人信息（无论年龄）。

7. 政策变更

如本隐私政策有任何更新，我们将在本页面更新"最后更新"日期。重大变更（例如开始收集数据）会通过扩展更新说明或官网公告告知用户。
